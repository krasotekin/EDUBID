# EDUBID


### 나라장터 공고 중 AI 관련 교육운영 용역을 필터링하고, 업스테이지 API를 활용해 공고 진단, 제안 작성 가이드, 초안 피드백을 제공하는 프로토타입입니다.


## 왜 만들었는가

업스테이지 Solar Pro 3 API 키를 발급받은 뒤, 제품 체험 차원에서 바이브코딩으로 간단한 프로토타입을 제작했습니다.

어떤 방식으로 활용해볼지 고민하던 중, AI를 이용해 영업 직무의 신입사원에게 실질적으로 도움이 될 수 있는 가이드라인과 피드백 제공 프로그램을 만들어보자는 방향을 잡았습니다.

제가 사업 개발과 영업 관련 대외활동을 하면서 가장 어렵게 느꼈던 부분은 *상대방의 니즈를 미리 파악하고, 그에 맞는 제안서를 준비*하는 일이었습니다. 바이어의 RFQ를 검토할 때도 단순히 제품의 특징만 정리하는 것으로는 부족했고, 상대가 실제로 무엇을 중요하게 보는지에 따라 제안의 방향과 성패가 크게 달라졌습니다. 또 실제 협상 과정에서는 우리가 예상한 니즈와 바이어의 실제 관심사가 달라 협상 전략을 처음부터 다시 조정해야 하는 경우도 있었습니다.

기존에는 이런 감각을 익히는 가장 좋은 방법이 결국 직접 협상 경험을 많이 쌓는 것이라고 생각했습니다. 하지만 이런 경험에서 축적되는 *판단 기준과 노하우 역시 일정 부분 구조화하고 자동화해, 신입사원에게 가이드라인 형태로 제공*할 수 있지 않을까 생각하게 되었고, 그 아이디어를 바탕으로 이번 프로토타입을 만들었습니다.

현재는 공공데이터 API로 받아올 수 있는 공공기관 입찰 및 견적 요청 정보를 기반으로, 제안 작성 가이드와 초안 피드백을 자동으로 제공하는 수준까지 구현했습니다. 아직은 초기 버전이지만, 만약 *기업 환경에서 활용한다면 과거 견적서, 입찰 결과, 제안 성공 및 실패 사례 등을 함께 학습*시켜 훨씬 더 정교하고 실무적인 영업 신입 가이드로 확장할 수 있을 것이라고 생각합니다.

## 핵심 기능

1. 나라장터 공고 조회  
   최근 공고를 불러오고, AI 관련 교육운영 성격이 있는 공고만 필터링합니다.

2. 공고 진단  
   선택한 공고를 바탕으로 업스테이지 API가 제안서 작성 시 중요한 포인트를 1~2문장으로 진단합니다.

3. 작성 가이드 생성  
   공고 유형과 진단 결과를 바탕으로  
   - 반드시 강조해야 할 항목  
   - 가격/예산 작성 시 주의점  
   - 유사실적 작성 팁  
   - 흔히 약해지는 포인트  
   를 정리해 줍니다.

4. 초안 피드백  
   사용자가 작성한 초안을 입력하면  
   - 잘한 점  
   - 빠진 점  
   - 수정 우선순위  
   - 실제 평가에서 약할 수 있는 이유  
   를 피드백합니다.

## 사용 기술

- LLM (Large Language Model)
- Prompt Engineering 
- RAG (Retrieval Augmented Generation)
- Python, REST API 
- Streamlit
- Upstage API
- 나라장터 OpenAPI
- XML parsing

## Upstage API를 어디에 사용했는가

이 프로젝트에서 업스테이지 API는 아래 3개 역할에 사용했습니다.

1. 공고 진단 생성  
   선택한 공고를 보고, 이 공고에서 제안서 작성 시 중요한 포인트를 1~2문장으로 진단

2. 작성 가이드 생성  
   공고명, 기관명, 공고 유형, 진단 결과를 바탕으로 제안 작성 방향 제시

3. 초안 피드백 생성  
   사용자의 초안을 공고 정보와 비교해 보완 포인트 제시

## 구현하면서 느낀 점

### 좋았던 점:
- 짧은 시간 안에 실제로 동작하는 프로토타입을 만들 수 있을 정도로 진입장벽이 낮았습니다.
- 단순 요약이 아니라 작성 가이드와 피드백 같은 업무형 출력으로 확장하기 쉬웠습니다.
- 공고 데이터와 결합했을 때 "문서를 읽고 바로 액션으로 연결하는" 흐름을 빠르게 시험해볼 수 있었습니다.

### 아쉬웠던 점:
- 이번 버전은 공고 목록과 제목/기관 정보를 중심으로 구현했기 때문에, 첨부된 제안요청서 PDF/HWP까지 직접 읽히면 훨씬 더 강력해질 수 있습니다.
- 실제 서비스 수준으로 가려면 문서 파싱, 구조화 추출, 평가항목 매핑까지 추가로 붙이는 것이 좋습니다.


## 실행 방법

#### 1. 프로젝트 폴더로 이동
cd ~/Desktop/edubid-guide

#### 2. 환경변수 설정 
프로젝트 루트에 .env 파일을 만들고 아래 값을 넣어야 합니다.

```sh
UPSTAGE_API_KEY=your_upstage_api_key
DATA_GO_KR_SERVICE_KEY=your_data_go_kr_service_key
```

#### 3. 가상환경 활성화
```sh
$ source .venv/bin/activate
```

#### 3. 패키지 설치
```sh
$ pip install -r requirements.txt
```

#### 4. 앱 실행
```sh
$ python -m streamlit run app.py
```
